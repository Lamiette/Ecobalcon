Tu travailles dans mon projet de site statique “EcoBalcon”, dédié au jardinage sur balcon, au potager urbain, aux plantes faciles, aux fleurs, aux petits espaces extérieurs et aux conseils pratiques pour débutants.

Objectif :
Créer et intégrer un nouvel article SEO complet, utile et cohérent avec le site, en choisissant toi-même le meilleur sujet après analyse du projet existant.

Tu dois agir comme un éditeur SEO senior + intégrateur de site statique :
- analyser les contenus déjà présents ;
- identifier une opportunité éditoriale pertinente ;
- créer un article naturel, utile et bien structuré ;
- l’intégrer proprement dans le projet ;
- mettre à jour uniquement les fichiers nécessaires ;
- vérifier que rien d’important n’est cassé.

────────────────────────
1. ANALYSE DU SITE
────────────────────────

Commence par analyser le projet existant :
- articles déjà publiés ;
- titres H1/H2/H3 ;
- sujets et mots-clés déjà couverts ;
- style rédactionnel ;
- structure HTML/Markdown utilisée ;
- métadonnées SEO ;
- maillage interne ;
- images utilisées ;
- sitemap, flux RSS, index d’articles, catégories/tags ou fichiers de données s’ils existent.

Identifie :
- les sujets déjà bien couverts ;
- les sujets incomplets ;
- les risques de doublon ou de cannibalisation ;
- les opportunités de longue traîne SEO ;
- les sujets adaptés à un lecteur français débutant/intermédiaire ;
- les sujets cohérents avec la saison si pertinent.

Choisis ensuite le meilleur sujet à créer maintenant.

Critères de choix :
- sujet précis, pas trop large ;
- utile pour un lecteur qui jardine sur balcon ;
- potentiel SEO réaliste ;
- complémentaire aux articles existants ;
- facile à mailler avec le contenu déjà publié ;
- pas de doublon évident avec un article existant.

Ne me demande pas de choisir le sujet. Décide en autonomie.

────────────────────────
2. NOTE DE DÉCISION
────────────────────────

Avant d’écrire l’article, crée ou ajoute dans ton récap final une courte note indiquant :
- sujet retenu ;
- mot-clé principal ;
- mots-clés secondaires ;
- intention de recherche ;
- raison du choix ;
- articles existants pertinents pour le maillage interne ;
- éventuels sujets écartés et pourquoi, si utile.

────────────────────────
3. RÉDACTION DE L’ARTICLE
────────────────────────

Rédige un article complet, original, naturel et utile.

Style attendu :
- français naturel ;
- ton clair, concret, rassurant et pédagogique ;
- pas de phrases creuses ;
- pas de blabla SEO artificiel ;
- pas de sur-optimisation ;
- conseils réellement exploitables ;
- cohérence avec le ton EcoBalcon.

Structure attendue :
- H1 naturel et optimisé ;
- introduction engageante ;
- sections H2/H3 bien organisées ;
- paragraphes lisibles ;
- listes à puces seulement si elles apportent de la clarté ;
- conseils pratiques ;
- erreurs à éviter si pertinent ;
- FAQ courte uniquement si elle apporte une vraie valeur SEO ;
- conclusion naturelle avec ouverture vers d’autres contenus du site.

La longueur doit être adaptée au sujet et au niveau des autres articles du site.
Ne gonfle pas artificiellement le texte.

────────────────────────
4. SEO ON-PAGE
────────────────────────

Optimise l’article avec :
- title SEO unique ;
- meta description naturelle et incitative ;
- slug propre ;
- structure Hn cohérente ;
- intégration naturelle du mot-clé principal ;
- variantes sémantiques ;
- alt text d’image ;
- canonical si la structure du site l’utilise ;
- Open Graph / Twitter cards si le projet les utilise ;
- schema.org Article ou BlogPosting si le site l’utilise déjà.

Priorité :
- intention de recherche ;
- lisibilité ;
- utilité réelle ;
- cohérence éditoriale ;
- probabilité de clic depuis Google.

N’invente pas de chiffres précis, de données botaniques douteuses ou de promesses non vérifiables.

────────────────────────
5. MAILLAGE INTERNE
────────────────────────

Ajoute des liens internes naturels :
- depuis le nouvel article vers les articles existants pertinents ;
- depuis 2 à 5 anciens articles vers le nouvel article si cela s’intègre naturellement.

Ne crée pas de liens forcés.
Ne modifie pas massivement d’anciens articles si ce n’est pas nécessaire.
Préserve le style des articles existants.

────────────────────────
6. IMAGE
────────────────────────

Prépare une image cohérente avec le sujet.

Règles :
- utiliser uniquement une image libre de droit avec source/licence vérifiable ;
- ne pas inventer de source ;
- éviter les images kitsch, trop artificielles ou hors sujet ;
- privilégier une image crédible pour un site de jardinage sur balcon ;
- nom de fichier optimisé SEO ;
- alt text naturel ;
- légende/crédit si le site en utilise.

Si l’environnement permet de télécharger et intégrer proprement l’image, fais-le.
Sinon, prépare l’intégration et laisse une note claire avec :
- source recommandée ;
- URL de l’image ;
- licence ;
- nom de fichier conseillé ;
- alt text ;
- emplacement prévu dans le projet.

────────────────────────
7. INTÉGRATION DANS LE SITE
────────────────────────

Crée réellement l’article dans le bon format du projet.
Respecte la structure existante.

Mets à jour uniquement les fichiers nécessaires, par exemple :
- page listing des articles ;
- page d’accueil si elle affiche les derniers articles ;
- sitemap ;
- flux RSS/Atom si présent ;
- fichiers de données JSON/YAML/JS/TS si utilisés ;
- pages catégories/tags si elles existent ;
- liens internes depuis anciens articles ;
- image preview sociale si le projet l’utilise.

Ne réorganise pas le projet.
Ne change pas la charte graphique.
Ne refactorise pas inutilement.

────────────────────────
9. CONTRÔLES QUALITÉ
────────────────────────

Avant de finaliser, vérifie :
- absence de doublon évident ;
- absence de cannibalisation SEO évidente ;
- cohérence du ton ;
- orthographe et grammaire ;
- liens internes valides ;
- métadonnées complètes ;
- image correctement intégrée ou documentée ;
- sitemap/RSS/index à jour si nécessaire ;
- rendu HTML cohérent avec les autres articles ;
- absence de modification inutile.

Si le projet contient des scripts de build, validation, lint ou preview, lance-les si possible.
Si une commande échoue, corrige si c’est raisonnable, sinon explique clairement le problème.

────────────────────────
10. RÉCAP FINAL
────────────────────────

À la fin, donne un récapitulatif clair avec :
- sujet choisi ;
- mot-clé principal ;
- intention de recherche ;
- raison du choix ;
- auteur utilisé ;
- fichiers créés ;
- fichiers modifiés ;
- éléments SEO ajoutés ;
- liens internes ajoutés ;
- image utilisée ou action manuelle restante ;
- commandes de vérification lancées ;
- éventuelles limites ou points à vérifier manuellement.

Règles importantes :
- décide en autonomie ;
- ne crée pas un sujet déjà traité sauf angle nettement différent ;
- ne rédige pas de contenu générique ;
- ne casse pas la structure existante ;
- fais des modifications propres, minimales et cohérentes ;
- privilégie la qualité réelle à la quantité ;
- si tu hésites, choisis le meilleur compromis entre utilité lecteur, cohérence éditoriale et potentiel SEO.
